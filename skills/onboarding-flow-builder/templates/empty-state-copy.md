# 빈 상태(empty state) 카피 — <product> <date>

빈 화면은 이탈 지점이다. "데이터 없음" 대신 **다음 행동 1개**를 제시한다.

## 빈 상태 3요소

각 빈 화면마다 채운다.

| 화면/리스트 | 1줄 가치 진술 | 단일 CTA | 샘플/시드 미리보기 |
|---|---|---|---|
| <홈 빈 목록> | <왜 여기 있어야 하는가, 한 줄> | <Aha 행동으로 직결되는 버튼 텍스트> | <샘플 1개 노출 여부> |
| <검색 0건> | <검색이 비었을 때 가치> | <대체 행동> | - |
| <첫 진입 대시보드> | ... | ... | ... |

## 카피 규칙

- 가치 진술은 1줄. 기능 설명 나열 금지.
- CTA는 화면당 1개. 동사로 시작 (예: "첫 노트 만들기", "사진 추가").
- 시드 데이터로 "완성된 모습"을 미리 보여주면 빈 화면 공포를 줄인다.

## 금지

- 클리니컬·과장 표현 (프로젝트 어휘 정책이 있으면 그 lexicon 따른다)
- 이모지·아이콘 장식
- 빈 화면에 설명 문단만 늘어놓기 (행동 없는 텍스트는 이탈)
- em dash, 과잉 multi-color

## 변형(A/B) 후보

| 변형 | 가설 | 측정 이벤트 |
|---|---|---|
| A (control) | 텍스트 CTA만 | onboarding_step_completed |
| B | 시드 데이터 미리보기 추가 | aha_moment_reached 도달 시간 단축 |

변형은 aha-moment-optimizer 실험 백로그에 등재한다.
