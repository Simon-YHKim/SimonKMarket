# lifecycle-campaign-designer

라이프사이클/CRM 캠페인을 설계하는 skill. growth-engine이 "어떻게 보내는가(채널·SDK)"라면, 이 skill은 그 위에 "**누구에게 · 언제 · 무엇을**"을 얹는다.

## 발동 조건

- "라이프사이클 캠페인 설계", "CRM 시퀀스 만들어줘", "윈백 캠페인"
- "리텐션 메시지", "재참여 캠페인", "온보딩 시퀀스", "이탈 방지 메시지"
- aarrr-growth-planner(Retention 단계)·growth-engine 이후 호출

## 경계 (위임 규칙)

| 이 skill | 위임 |
|---|---|
| 세그먼트·트리거·메시지·시퀀스 설계 | 발송 SDK/도메인 인증/딥링크 → `growth-engine` |
| 캠페인별 측정 지표·holdout 설계 | 이벤트 정의·퍼널 추적 → `analytics-integrator` |
| 행동 세그먼트의 "라이프사이클 단계" 명세 | 코호트 리텐션 곡선 산출 → `cohort-retention-analyzer` |

발송 배관이 아직 없으면 먼저 `growth-engine`을, 코호트 데이터가 없으면 `cohort-retention-analyzer`를 선행할 것.

## 진단 먼저 (AskUserQuestion)

설계 전 확인. 답이 없으면 추측하지 말고 묻는다.

1. **제품 유형** — 구독 SaaS / 트랜잭션 커머스 / 무료+광고 / 프리미엄. (수익 모델에 따라 윈백 트리거가 달라짐)
2. **핵심 가치 행동(activation event)** — 사용자가 "이걸 했으면 남는다"는 단일 행동. (예: 첫 노트 저장, 첫 결제, 3일 연속 접속)
3. **데이터·발송 기반** — 이벤트 추적 있나? 푸시/이메일/인앱 중 무엇이 살아있나?
4. **동의·연락처 보유율** — 이메일 동의 / 푸시 권한 / 마케팅 수신 동의 비율 (채널 선택의 상한선)

## 1단계 — 세그먼트 (RFM + 행동)

두 축을 병행한다. RFM은 "가치", 행동 단계는 "맥락".

### RFM 세그먼트 (커머스/구독 가치 분류)

| 세그먼트 | R(최근성) | F(빈도) | M(금액) | 액션 |
|---|---|---|---|---|
| Champions | 높음 | 높음 | 높음 | 리워드·추천 유도, 광고 holdout |
| Loyal | 중간 | 높음 | 중간+ | 업셀·교차판매 |
| Potential Loyalist | 높음 | 중간 | 중간 | 습관화 넛지 |
| At-Risk | 낮음 | 높음(과거) | 높음(과거) | **윈백 우선순위 1** |
| Hibernating | 매우 낮음 | 낮음 | 낮음 | 저비용 재참여(이메일만) |
| Lost | 최저 | 최저 | 최저 | 분기 1회 브로드캐스트, 그 외 제외 |

> 임계값(R/F/M 분위)은 제품 데이터로 정한다. 근거 없는 "30일=이탈" 단정 금지 — `cohort-retention-analyzer` 곡선에서 생존율이 꺾이는 지점을 임계값으로 쓴다.

### 행동(라이프사이클) 단계

```
신규(D0) → 활성화 전(activation 미달) → 활성 → 습관 → 이탈조짐 → 휴면 → 이탈
```

각 단계는 "관측 가능한 이벤트"로 정의해야 한다(추측 금지).
- 활성화 전: 가입 O, activation event X
- 이탈조짐: 직전 7일 사용량 < 개인 기준선의 N% (감소 추세)
- 휴면: 미사용 일수 ≥ 코호트 곡선의 churn 임계값

## 2단계 — 세그먼트 × 트리거 × 메시지 매트릭스

설계의 핵심 산출물. 한 행 = 한 캠페인.

| 세그먼트 | 트리거(이벤트/스케줄) | 채널(우선순위) | 메시지 의도 | 1차 KPI |
|---|---|---|---|---|
| 신규(활성화 전) | 가입 후 +1h, activation 미발생 | 푸시 → 인앱 → 이메일 | activation event로 1스텝 안내 | activation rate |
| 신규(활성화 전) | 가입 후 D1, 여전히 미발생 | 이메일 | "이것부터 해보세요" 단일 CTA | D1 activation |
| 활성 | activation 직후 | 인앱 | 다음 가치 행동 소개 | feature adoption |
| 이탈조짐 | 사용량 -50% 감지 | 푸시 → 이메일 | 개인화된 "이어서 하기" | 사용 재개율 |
| 휴면 | 미사용 D14 | 이메일(저비용) | 그동안의 변화/신기능 | 재방문율 |
| At-Risk(구독) | 결제 D-3 + 사용량 저조 | 이메일 + 인앱 | 가치 리마인드(취소 방어 아님) | 갱신율 |
| 구독 취소 | 취소 즉시 | 인앱 설문 → 이메일 | 이유 수집 + 조건부 혜택 | 윈백율 |
| 결제 실패 | 실패 D1/D3/D7 | 이메일 + 인앱 | Dunning(결제수단 갱신) | 회수율 |

트리거 종류: **이벤트 기반**(가입·구매·실패·활성화) / **부재 기반**(미사용 N일·미활성화) / **스케줄 기반**(주기 리포트·갱신 D-N). 부재 기반은 이벤트 부재를 감지하는 별도 잡이 필요하므로 `analytics-integrator`와 정의를 맞춘다.

## 3단계 — 표준 시퀀스 3종

빈 슬롯 채우기용 템플릿. 실제 카피·딜레이·혜택은 제품 맥락으로 채운다.
`templates/campaign-matrix.csv`에 머신리더블 버전, `templates/sequence-templates.md`에 단계별 카피 골격이 있다.

### A. 온보딩 (신규 → 활성화)
- 목표: activation event 도달. 메시지 수가 아니라 **단일 다음 행동**에 집중.
- 부정 트리거: activation 발생 시 시퀀스 즉시 중단(suppression).

### B. 재참여 (활성 → 이탈조짐)
- 목표: 사용량 감소를 조기에 되돌림. 휴면 전에 잡는 게 윈백보다 싸다.
- 개인화: 마지막으로 만지던 객체/진행 상태를 핵심 후크로.

### C. 윈백 (휴면/이탈 → 복귀)
- 목표: 복귀 1회 + 재활성화. "혜택 남발"은 마진과 학습된 이탈을 부른다.
- 단계: ①가치 리마인드(혜택 X) → ②신규/개선점 → ③조건부 혜택(마지막) → 무반응 시 빈도 다운그레이드 후 종료.

## 4단계 — 채널 선택

| 채널 | 강점 | 한계 | 권장 용도 |
|---|---|---|---|
| 푸시 | 즉시성·무료 | 권한 옵트인 필요, 피로 빠름 | 시간 민감 트리거, 활성 사용자 |
| 이메일 | 저비용·긴 콘텐츠·딥링크 | 도달·개봉률 낮음 | 휴면/윈백, 긴 설명, 영수증 |
| 인앱 | 맥락 100%·권한 불필요 | 앱에 들어와야만 노출 | 활성 세션 중 넛지, 업셀 |

규칙:
- 한 사용자에게 **동시 다채널 중복 발송 금지**(우선순위 폴백: 푸시 실패/미옵트인 → 이메일).
- frequency cap: 사용자당 하루 N개·주 M개 상한. 단계 간 최소 딜레이 보장.
- quiet hours(KST 야간) 적용 — 푸시는 특히.
- 동의 없는 마케팅 발송 금지(아래 컴플라이언스).

## 5단계 — Holdout & 측정

캠페인 효과는 **보낸 사람 vs 안 보낸 사람**의 차이(incremental lift)로 본다. 개봉률·클릭률은 보조 지표일 뿐.

- **Universal holdout**: 전체의 5~10%는 모든 라이프사이클 메시지에서 영구 제외 → 프로그램 전체의 순증분 측정.
- **Campaign holdout**: 각 캠페인 대상의 일부를 무처치 대조군으로.
- **1차 지표 = 행동/매출**, 2차 = 개봉·클릭. lift = (처치군 전환 − 대조군 전환).
- 최소 표본: MDE 기반 계산. 표본 부족하면 캠페인을 합치거나 기간을 늘린다.
- 측정 보조: `scripts/holdout-lift.ts`로 처치/대조 카운트에서 lift·상대증분·근사 유의성 계산.

이벤트 정의·대시보드 연동은 `analytics-integrator`로 위임하되, 캠페인마다 1차 KPI를 매트릭스에 못박아 둔다.

## 컴플라이언스 (한국 + 글로벌)

- **수신 동의 필수**: 마케팅 메시지는 사전 동의(opt-in). 영리 목적 야간(21~08시) 발송은 별도 동의 — 정보통신망법.
- 광고성 메시지: 제목/본문에 "(광고)" 표기, 무료 수신거부 수단 명시.
- **수신거부 즉시 반영**: unsubscribe/푸시 옵트아웃은 모든 후속 시퀀스에서 즉시 제외.
- 14~17세·미성년 대상은 발송 전 연령/동의 상태 확인(프로젝트 C10류 제약 준수).
- 거래성(영수증·결제실패·보안) 메시지는 광고가 아니므로 별도지만, 마케팅을 끼워넣지 말 것.

## 산출물

1. 세그먼트 정의표(RFM 임계값 + 행동 단계 이벤트 명세)
2. 세그먼트 × 트리거 × 메시지 매트릭스 (`templates/campaign-matrix.csv`)
3. 온보딩/재참여/윈백 시퀀스 3종 (`templates/sequence-templates.md`)
4. 채널 매핑 + frequency cap + quiet hours 정책
5. holdout 설계 + 캠페인별 1차 KPI + 측정 방법

## 검증 체크리스트

- [ ] 모든 세그먼트가 "관측 가능한 이벤트"로 정의됨 (추측·매직넘버 없음)
- [ ] 모든 캠페인 행에 1차 KPI가 명시됨
- [ ] activation/구매 발생 시 해당 시퀀스 suppression 동작
- [ ] 다채널 중복 발송 방지 + frequency cap 적용
- [ ] universal holdout(5~10%) 확보 — lift 측정 가능
- [ ] 마케팅 발송 전 동의 상태 확인, 수신거부 즉시 반영
- [ ] quiet hours / 야간 발송 동의 분리
