# idea-validation

출시 전, 코드 한 줄 쓰기 전에 "이 아이디어가 실제 수요가 있는가"를 저비용으로 검증하는 skill.

## 발동 조건

- "이 아이디어 검증해줘", "이거 될까?", "수요 있을까?"
- "출시 전에 확인하고 싶어", "테스트해보고 싶어"
- "validate this idea", "demand test", "will this work"
- 아직 만들지 않았거나 만들기 시작 직전 단계

## 핵심 원칙

- **만들기 전에 판다.** 코드보다 대화·랜딩·사전판매가 먼저다.
- **저자산 우선.** 1인/저예산 페르소나용. 광고비 0~5만원, 백엔드 0개로 가능한 방법만 권장.
- **거짓 신호 거른다.** "좋은 아이디어네요" 칭찬은 데이터가 아니다. 행동(돈·시간·이메일)만 신호로 친다.
- **빠른 kill 이 승리다.** 안 될 아이디어를 1주 안에 죽이는 게, 6개월 만들고 죽이는 것보다 낫다.

## Workflow

### 0. 인터뷰 (AskUserQuestion)

아래를 모르면 먼저 묻는다 (한 번에 2~3개씩):

- **문제**: 누구의 어떤 불편을 해결하나?
- **타겟**: 가장 아파하는 한 명(beachhead)은 누구인가?
- **현재 대안**: 그 사람은 지금 이 문제를 어떻게 때우고 있나? (대체재)
- **가설 위험도**: "사람들이 원할까(수요)"가 불안한가, "만들 수 있을까(실현)"가 불안한가? → 전자면 이 skill 이 맞다.
- **자산**: 쓸 수 있는 예산·시간·기존 청중(팔로워/메일링)은?

### 1. 문제 가설 정의

한 문장 템플릿으로 강제 구체화:

```
[타겟 고객] 은 [상황] 에서 [문제] 때문에 [현재 대안] 을 쓰지만
[불만/비용] 을 겪고 있다. 우리는 [해결 가설] 으로 이를 없앤다.
```

검증할 가설을 위험도 순으로 분해:

| 가설 유형 | 질문 | 이 skill 의 검증 도구 |
|---|---|---|
| 문제 가설 | 이 문제가 진짜 아픈가? | 고객 인터뷰 (2단계) |
| 수요 가설 | 돈/이메일/시간을 낼 만큼 원하나? | 수요 테스트 (3단계) |
| 차별화 가설 | 기존 대안보다 나은가? | 경쟁·대체재 매핑 (4단계) |
| 규모 가설 | 먹고 살 만큼 큰 시장인가? | TAM/SAM/SOM (5단계) |

> 가장 불안한 가설부터 가장 싼 방법으로 친다. 모든 가설을 다 검증하려 하지 말 것.

### 2. 고객 인터뷰 설계 — The Mom Test

목적: **솔루션이 아니라 문제를 검증**한다. "내 엄마한테 물어도 거짓말이 안 나오는 질문"만 한다.

Mom Test 3원칙:
1. 내 아이디어가 아니라 **상대의 삶**을 말하게 한다.
2. **미래 가정("쓰실래요?")이 아니라 과거 사실("최근에 어떻게 하셨어요?")**을 묻는다.
3. **칭찬·의견을 구하지 않는다.** 행동과 구체적 사례를 캔다.

좋은 질문 vs 나쁜 질문:

| 나쁜 질문 (거짓 신호) | 좋은 질문 (사실 캐기) |
|---|---|
| "이런 앱 있으면 쓰실래요?" | "마지막으로 이 문제 겪은 게 언제예요? 그때 어떻게 하셨어요?" |
| "월 9,900원이면 살 만해요?" | "지금 이거 해결하려고 돈/시간 얼마나 쓰세요?" |
| "좋은 아이디어 같죠?" | "이 문제 때문에 뭘 시도해봤다 실패했어요?" |
| "기능 A 어때요?" | "그 작업 중에 가장 짜증나는 단계가 어디예요?" |

진행 가이드:
- 표본: 타겟 세그먼트 5~10명. 지인 말고 **진짜 그 문제를 겪는 사람**.
- 모집: 커뮤니티(카페/디스코드/오픈채팅/레딧), DM, 지인의 지인.
- 신호 판독:
  - 강함 → 이미 돈/시간 들여 자체 해결(엑셀·수동·경쟁 툴) 중. "지금 당장 베타 달라."
  - 약함 → "좋네요" 칭찬만, 과거 행동 사례 없음, 연락처 안 줌.

### 3. 수요 테스트 — 저비용 3종

말이 아닌 **행동**을 사는 단계. 자산 수준에 맞춰 1개 이상 고른다.

| 방법 | 측정 신호 | 비용/시간 | 언제 |
|---|---|---|---|
| 랜딩 + 대기자명단 | 방문→이메일 등록 전환율 | ₩0~5만, 1~2일 | 수요 자체가 불확실 |
| 스모크 테스트(가짜 문) | "구매/시작" 클릭률 (실제 결제 X) | ₩0~5만, 1일 | 어떤 기능/가격에 반응하나 |
| 사전판매 | 실제 결제·예약금 | ₩0, 며칠 | 가장 강한 신호 — 돈이 움직임 |

**랜딩 + 대기자명단**
- 1페이지면 충분: 헤드라인(문제→약속) + 3개 베네핏 + 이메일 1칸 CTA.
- 도구: Carrd / Framer / Vercel + Tally·Google Form (백엔드 불필요).
- 트래픽: 인터뷰했던 커뮤니티에 직접 공유 + 소액 광고(선택).
- 합격선(러프): 콜드 트래픽 이메일 전환 **>10%**, 타겟 커뮤니티면 **>20%**.

**스모크 테스트 (fake door)**
- 아직 없는 기능/플랜 버튼을 노출 → 클릭 시 "곧 출시, 알림 받기".
- 가격 민감도 테스트: 가격 다른 2개 랜딩으로 A/B.

**사전판매**
- "정식 출시 전 50% 얼리버드" 예약/결제. 환불 보장으로 위험 제거.
- 결제 1건 = 인터뷰 100마디보다 강하다. 0건이면 무료 관심은 가짜였다는 뜻.

> 핸드오프: 랜딩을 제대로 만들 단계가 되면 `design-consultation` → `design-html` 로 넘긴다. 이 skill 은 "검증용 1페이지"까지만 설계한다.

### 4. 경쟁 · 대체재 매핑

WebSearch / WebFetch 로 조사한다. "경쟁자 없음"은 위험 신호(시장 없음일 수 있음).

조사 항목:
- 직접 경쟁자(같은 솔루션), 간접 경쟁자(다른 솔루션 같은 문제).
- **진짜 대체재**: 엑셀, 수기, 카톡, "그냥 참기", 사내 툴 — 대부분 진짜 적은 이쪽.
- 각각의 가격·강점·약점·리뷰의 불만(앱스토어·커뮤니티에서 "별 1~2점" 캐기).

산출 포맷:

| 대안 | 유형 | 가격 | 강점 | 약점(=내 기회) |
|---|---|---|---|---|

차별화 점검:
- 내 해결책이 기존 대비 **10배 낫거나, 절반 가격이거나, 새 세그먼트**를 여나? 셋 다 아니면 차별화 가설 실패.

### 5. 시장 규모 가늠 — TAM / SAM / SOM (러프)

정밀 추정 아님. **자릿수(order of magnitude)** 확인용.

```
TAM = 전체 시장 (이 문제 가진 전체 인구 × 연간 지불액)
SAM = 내가 도달 가능한 세그먼트 (지역·언어·채널 제약 반영)
SOM = 1~2년 내 현실적으로 먹을 몫 (SAM의 1~5%)
```

Bottom-up 우선(신뢰도↑):
```
SOM = (도달 가능한 채널의 잠재 고객 수) × (현실적 전환율) × (객단가)
예: 타겟 커뮤니티 5만명 × 가입 2% × 결제 10% × 연 12만원 = 1,200만원/년
```

- WebSearch 로 인구·시장 보고서 자릿수만 확보. 출처 명시.
- 판독: SOM 이 1인 개발자 생존선(예: 연 매출 목표) 자릿수에 미달하면 규모 가설 실패.

### 6. Go / Pivot / Kill 판단

신호를 모아 점수화. 각 가설 PASS/FAIL:

| 가설 | PASS 기준(러프) |
|---|---|
| 문제 | 인터뷰 5+명 중 과반이 "이미 돈/시간 써서 해결 시도" |
| 수요 | 랜딩 전환 >10%(콜드) 또는 사전판매 1건+ |
| 차별화 | 10배 낫거나 / 절반 값 / 새 세그먼트 중 1개 충족 |
| 규모 | SOM 자릿수가 목표 매출 충족 |

판정:

| 결과 | 조건 | 다음 행동 |
|---|---|---|
| **GO** | 4개 중 3+ PASS, 수요 가설 필수 PASS | MVP 착수 → `app-dev-orchestrator` / `simon-tdd` |
| **PIVOT** | 문제는 PASS, 수요·차별화·규모 중 FAIL | 세그먼트·솔루션·가격 중 1축만 바꿔 재검증 |
| **KILL** | 문제 가설 FAIL 또는 수요 0 신호 | 아카이브. 인터뷰에서 들은 "옆 문제"를 새 가설로 |

### 7. 산출물

다음을 정리해 사용자에게 반환(파일은 사용자가 원할 때만):

```
1. 문제 가설 한 문장
2. Mom-Test 인터뷰 스크립트 (질문 5~8개)
3. 수요 테스트 플랜 (선택한 방법 + 합격선 + 도구)
4. 경쟁·대체재 표
5. TAM/SAM/SOM 러프 (출처 포함)
6. Verdict: GO / PIVOT / KILL + 근거 + 다음 행동
```

## 검증 체크리스트

- [ ] 가장 불안한 가설부터 가장 싼 방법으로 쳤는가
- [ ] 인터뷰 질문이 미래 가정이 아닌 과거 사실을 묻는가 (Mom Test)
- [ ] 수요 신호가 "말"이 아니라 "행동"(이메일·클릭·결제)인가
- [ ] "경쟁자 없음"을 기회로 착각하지 않았는가
- [ ] 시장 규모를 bottom-up 으로 자릿수까지 확인했는가
- [ ] Verdict 가 데이터에 근거하는가, 애착에 근거하는가
